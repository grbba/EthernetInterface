\doxysubsection{Detailed Description}
Extended stream interface for terminal operations. 

This class extends the standard Arduino Stream class with functions that are suitable for interfacing to V\+T100 terminal applications like Pu\+T\+TY.

The following example initializes a terminal running on the primary Arduino serial port\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{class_terminal}{Terminal}} term;}
\DoxyCodeLine{\textcolor{keywordtype}{void} setup() \{}
\DoxyCodeLine{    \mbox{\hyperlink{class_terminal_a6fd7d021468962e9604bee0522b3a364a7efe0e37dae166c00d913190cd8f55f6}{Serial}}.begin(9600);}
\DoxyCodeLine{    term.\mbox{\hyperlink{class_terminal_a6bf417195a7d2a2fab6b3518f25764a2}{begin}}(\mbox{\hyperlink{class_terminal_a6fd7d021468962e9604bee0522b3a364a7efe0e37dae166c00d913190cd8f55f6}{Serial}});}
\DoxyCodeLine{\}}
\end{DoxyCode}


The \mbox{\hyperlink{class_terminal_aa82621ea174422601cc7384c8185eb96}{read\+Key()}} function reads input from the underlying stream, decodes any V\+T100 key escape sequences that it finds, and reports them to the application using U\+SB, A\+S\+C\+II, or Unicode key codes. This is typically used in the application\textquotesingle{}s main loop as follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{void} loop() \{}
\DoxyCodeLine{    \textcolor{keywordtype}{int} key = term.\mbox{\hyperlink{class_terminal_aa82621ea174422601cc7384c8185eb96}{readKey}}();}
\DoxyCodeLine{    \textcolor{keywordflow}{switch} (key) \{}
\DoxyCodeLine{    \textcolor{keywordflow}{case} -\/1:     \textcolor{keywordflow}{break};      \textcolor{comment}{// No key available.}}
\DoxyCodeLine{}
\DoxyCodeLine{    \textcolor{keywordflow}{case} \mbox{\hyperlink{_u_s_b_keys_extra_8h_a9a8100ac50bf8e0227c66f85312d1eec}{KEY\_LEFT\_ARROW}}:}
\DoxyCodeLine{        \textcolor{comment}{// Left arrow key has been pressed.}}
\DoxyCodeLine{        ...}
\DoxyCodeLine{        \textcolor{keywordflow}{break};}
\DoxyCodeLine{}
\DoxyCodeLine{    \textcolor{keywordflow}{case} \mbox{\hyperlink{_u_s_b_keys_extra_8h_a6b1df3d1cbf22d04b162eb6a89068b5d}{KEY\_RIGHT\_ARROW}}:}
\DoxyCodeLine{        \textcolor{comment}{// Right arrow key has been pressed.}}
\DoxyCodeLine{        ...}
\DoxyCodeLine{        \textcolor{keywordflow}{break};}
\DoxyCodeLine{}
\DoxyCodeLine{    \textcolor{keywordflow}{case} \mbox{\hyperlink{_u_s_b_keys_extra_8h_a86b2b2d4eda22067d85d2287c118e2d8}{KEY\_ESC}}:}

\DoxyCodeLine{            \textcolor{comment}{// Visible ASCII character has been typed.}}

This class understands extended characters in the U\+T\+F-\/8 encoding, allowing the full Unicode character set to be used in applications. Extended Unicode characters are reported by \mbox{\hyperlink{class_terminal_aa82621ea174422601cc7384c8185eb96}{read\+Key()}} with the key code K\+E\+Y\+\_\+\+U\+N\+I\+C\+O\+DE, with the actual code point returned via \mbox{\hyperlink{class_terminal_ad0b704240c300615defdc76d33d5cac2}{unicode\+Key()}}.

On the output side, U\+T\+F-\/8 strings can be written to the terminal using \mbox{\hyperlink{class_terminal_ae723f16d4f97954fb45550fa05d600bb}{write()}}, or \mbox{\hyperlink{class_terminal_a25308d6b2a3d18820d8a0e641c7be937}{write\+Unicode()}} can be used to write a single Unicode character in U\+T\+F-\/8.

\begin{DoxyNote}{Note}
This class does not have any special support for right-\/to-\/left scripts or composed characters. Unicode characters are read and written in the order in which they arrive. Applications may need to alter strings to display them correctly in such scripts. Patches are welcome to fix this.
\end{DoxyNote}

\doxysubsubsection{\texorpdfstring{read()}{read()}}
{\footnotesize\ttfamily int Terminal\+::read (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}



Reads the next byte from the underlying stream. 

\begin{DoxyReturn}{Returns}
Returns 0x00 to 0x\+FF if a byte is ready, or -\/1 if none available.
\end{DoxyReturn}
This function performs a low-\/level read on the underlying byte stream without applying any specific interpretation to the byte. In particular, escape sequences corresponding to arrow and function keys will not be recognized.

Applications will usually want to call \mbox{\hyperlink{class_terminal_aa82621ea174422601cc7384c8185eb96}{read\+Key()}} instead to handle escape sequences for arrow and function keys. This function is provided as a convenience to implement the parent Stream interface.

\doxysubsubsection{\texorpdfstring{readKey()}{readKey()}}
{\footnotesize\ttfamily int Terminal\+::read\+Key (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}



Reads the next key that was typed on this terminal. 

\begin{DoxyReturn}{Returns}
Returns -\/1 if there is no key ready yet; 0x00 to 0x7F for an A\+S\+C\+II character; K\+E\+Y\+\_\+\+U\+N\+I\+C\+O\+DE for an extended Unicode code point, or a U\+SB keyboard code for special arrow or function keys.
\end{DoxyReturn}
For example, if the user types the Home key, then this function will return K\+E\+Y\+\_\+\+H\+O\+ME. If the user types the capital letter A, then this function will return 0x41.

If the user types an extended Unicode character (U+0080 and higher), then this function will return K\+E\+Y\+\_\+\+U\+N\+I\+C\+O\+DE. The application should call \mbox{\hyperlink{class_terminal_ad0b704240c300615defdc76d33d5cac2}{unicode\+Key()}} to retrieve the actual code point. All Unicode characters are assumed to be in the U\+T\+F-\/8 encoding on the stream.

Some A\+S\+C\+II control characters correspond to special keys and will be mapped appropriately\+:

\begin{DoxyItemize}
\item 0x08 (C\+T\+R\+L-\/H) and 0x7F (D\+EL) are mapped to K\+E\+Y\+\_\+\+B\+A\+C\+K\+S\+P\+A\+CE \item 0x0D (C\+T\+R\+L-\/M) and 0x0A (C\+T\+R\+L-\/J) are mapped to K\+E\+Y\+\_\+\+R\+E\+T\+U\+RN \item 0x09 (C\+T\+R\+L-\/I) is mapped to K\+E\+Y\+\_\+\+T\+AB \item 0x1B (C\+T\+R\+L-\/\mbox{[}) is mapped to K\+E\+Y\+\_\+\+E\+S\+C\+A\+PE\end{DoxyItemize}
In all of these cases, the original A\+S\+C\+II code will be reported by \mbox{\hyperlink{class_terminal_ad0b704240c300615defdc76d33d5cac2}{unicode\+Key()}}. As a special case, if 0x0D is immediately followed by 0x0A (that is, C\+R\+LF) then K\+E\+Y\+\_\+\+R\+E\+T\+U\+RN will be reported only once with \mbox{\hyperlink{class_terminal_ad0b704240c300615defdc76d33d5cac2}{unicode\+Key()}} set to 0x0D. This ensures that all line ending types are mapped to a single K\+E\+Y\+\_\+\+R\+E\+T\+U\+RN report.

If the window size has changed due to a remote event, then K\+E\+Y\+\_\+\+W\+I\+N\+S\+I\+ZE will be returned. This can allow the caller to clear and redraw the window in the new size.

{\footnotesize\ttfamily long Terminal\+::unicode\+Key (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption}) const\hspace{0.3cm}{\ttfamily [inline]}}



Gets the Unicode version of the last key returned by \mbox{\hyperlink{class_terminal_aa82621ea174422601cc7384c8185eb96}{read\+Key()}}. 

If \mbox{\hyperlink{class_terminal_aa82621ea174422601cc7384c8185eb96}{read\+Key()}} returned an A\+S\+C\+II character (0x00 to 0x7F) or K\+E\+Y\+\_\+\+U\+N\+I\+C\+O\+DE, then this function can be used to query the full Unicode code point for the key that was typed. If some other key is typed, or no key was typed, then this function will return -\/1.

Unicode code points range between 0 and 0x10\+F\+F\+FF.

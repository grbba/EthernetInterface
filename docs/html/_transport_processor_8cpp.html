Sending a reply by using the StringFormatter (this will result in every byte send individually which may/will create an important Network overhead).

Here we hook back into the DCC code for actually processing the command using a DCCParser. Alternatively we could use MemeStream in order to build the entiere reply before ending it (cf. Scratch pad below)

Parameters
streamActually the Client to whom to send the reply. As Clients implement Print this is working
commandThe reply to be send ( echo as in sendReply() )
blockingif set to true will instruct the DCC code to not use the async callback functions

Set the App Protocol.

The detection id done upon the very first message recieved. The client will then be bound to that protocol. Its very brittle as e.g. The N message as first message for WiThrottle is not a requirement by the protocol; If any client talking Withrottle doesn't implement this the detection will default to JMRI. For HTTP we base this only on a subset of th HTTP verbs which can be used.

Parameters
aFirst character of the recieved buffer upon first connection
bSecond character of the recieved buffer upon first connection
Returns
appProtocol
